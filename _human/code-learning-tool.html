How Expert Programmers Learn Code 專家學習法

Many novice programmers default to reading source code as if it were a novel, or relying on AI generation to bypass the process of understanding. When faced with a complex system, they copy-paste snippets until it "works," leaving them unable to fix issues when the code breaks.

許多新手工程師在閱讀程式碼時，常像讀小說般一掃而過，或過度依賴 AI 生成來規避理解的過程。當面對複雜系統時，他們盲目複製貼上程式碼直到「可行」為止，一旦系統損壞便束手無策。

Expert programmers approach code with a completely different mindset. They prioritize building stable mental models, tracing flow manually, and treating libraries or runtimes as transparent tools. Here is how they master code autonomously:

專家級工程師則以截然不同的思維看待程式碼。他們優先建立穩固的心理模型、手動追蹤流程，並將函式庫或執行期環境視為透明的工具。以下是他們自主掌握程式碼的方法：

1. Tracing the Lifecycle of State (Mental CPU Execution) 1. 追蹤狀態生命週期（大腦模擬 CPU 執行）

Instead of just looking at functions in isolation, experts trace code by acting as the interpreter. In reactive UI systems (like React), this means tracing the lifecycle of state updates and matching them to rendering boundaries.

專家不會孤立地看各個函式，而是扮演直譯器來追蹤程式碼。在響應式 UI 系統（如 React）中，這意味著追蹤狀態更新的生命週期，並將其與渲染邊界進行對應。

Mental Trace Example: Language Switching 狀態追蹤範例：切換語言

Let's trace how Encounter Cards switches language from Chinese to Bilingual:
1. User clicks the "中/EN" button in the Setup screen.
2. This triggers the chooseLanguage('bilingual') callback in App.tsx.
3. Inside this callback, setLanguage('bilingual') is called, scheduling a React state update, and saveLanguage('bilingual') is called, writing the key encounter-language to LocalStorage.
4. On the next render pass, language resolves to 'bilingual'.
5. CardText receives language="bilingual". Its logic runs: language !== 'en' renders Chinese, and language !== 'zh' renders English. Both are rendered on screen simultaneously!

讓我們追蹤相遇卡如何將語言設定從中文切換為雙語：
1. 使用者在設定面板點擊「中/EN」按鈕。
2. 這觸發了 App.tsx 中的 chooseLanguage('bilingual') 回呼函式。
3. 在該函式中，呼叫 setLanguage('bilingual') 安排 React 狀態更新，並呼叫 saveLanguage('bilingual') 將值寫入 LocalStorage。
4. 在下一次渲染時，language 解析為 'bilingual'。
5. CardText 接收到 language="bilingual"。其邏輯開始運算：language !== 'en' 渲染中文段落，且 language !== 'zh' 渲染英文段落。中英雙語段落因此同時呈現在螢幕上！

2. Deconstruction and Reconstruction (The MVP Method) 2. 解構與重構（最小可行性產品法）

Experts never try to write a complex 10-module system all at once. They split the target into its absolute minimum functional components, verify each block, and grow it step-by-step.

專家絕不會試圖一次寫完包含 10 個模組的複雜系統。他們會將目標拆解為最基礎的功能組件，驗證每個區塊，並逐步擴展。

  1. Step 1: Raw Data & Filtering: Create an array of cards and write a clean function (like eligibleCards) to filter them.
  2. 步驟 1：原始資料與篩選： 建立卡牌陣列並寫一個乾淨的篩選函式（如 eligibleCards）來過濾它們。
  3. Step 2: Simple Rendering: Build a basic HTML container that renders the active card's text.
  4. 步驟 2：簡單渲染： 建立一個基本的 HTML 容器，將當前選中卡牌的文字渲染出來。
  5. Step 3: State & Actions: Add a "Draw" button. Implement nextCard logic to pick cards randomly while avoiding direct repetitions.
  6. 步驟 3：狀態與動作： 加入「抽卡」按鈕。實作 nextCard 邏輯以隨機選取卡牌且不重複。
  7. Step 4: Layout & Emulation: Implement the 430 × 932 container scaling rules in CSS and JS to ensure mobile proportions work correctly on desktop.
  8. 步驟 4：佈局與模擬： 在 CSS 和 JS 中實作 430 × 932 容器縮放規則，確保行動端比例在桌面上正確顯示。
  9. Step 5: Gestures & Refinement: Layer on pointer drag listeners to trigger the draw step, and write canvas drawers to assemble exportable files.
  10. 步驟 5：手勢與細節： 疊加指標拖曳監聽器來觸發抽卡動作，並撰寫 canvas 繪圖程式來輸出圖檔。

3. Rubber Duck Debugging (Feynman Technique) 3. 小黃鴨除錯法（費曼學習法）

When a bug appears, experts do not blindly guess fixes. They explain the code out loud line-by-line to a colleague, a rubber duck, or themselves. For example, explaining the hash-mapping function in deity-art.ts:

當遇到 Bug 時，專家不會盲目猜測修復方案。他們會對著同事、一隻小黃鴨或自己，逐行大聲解釋程式碼運作邏輯。例如，解釋 deity-art.ts 中的雜湊對應： 

// We want cards to always show the same deity artwork without storing a database relationship.
// To do this, we treat the card's ID string as a seed.
// We map over every character, multiplying the accumulated hash by 31 and adding character codes.
// Modulo the hash against the count of deity artworks. This yields a deterministic, offline index!

By describing *why* a function operates rather than *what syntax* it uses, developers expose logical assumptions and find the exact boundary where reality diverges from expectation.

藉由描述函式運作的「原因」而非「語法」，開發者能暴露其邏輯假設，並找出程式行為偏離預期的確切邊界。

4. Deep Autonomy from AI 4. 培養脫離 AI 的自主能力

AI is a useful autocomplete, but a terrible architect. Relying on AI prevents a developer from forming the cognitive pathways required to hold code structures in memory.

AI 是好用的自動完成工具，但卻是個糟糕的架構師。過度依賴 AI 會阻止開發者建立在記憶中建構程式碼結構所需的認知路徑。

  • Read the errors: When compilation fails, read the stack trace. The compiler is telling you the exact syntax rule or type contract that was violated.
  • 閱讀錯誤： 追蹤日誌報錯。
  • Verify boundaries: Write unit tests. Codebases like Encounter Cards include automated tests for age gates, scaling math, and random selection. Running these tests verifies modifications instantly without spinning up a browser.
  • 驗證邊界： 撰寫單元測試。像相遇卡這樣的程式庫包含年齡門檻、縮放計算和隨機抽選的測試。執行測試可直接驗證變更，而不需要開啟瀏覽器。
  • Inspect with developer tools: Use the browser's Inspect Element and console logger (F12) to audit layout scaling and local storage keys instead of guessing why styling fails.
  • 利用開發者工具檢查： 使用瀏覽器的「檢查元素」與主控台日誌（F12）來稽核縮放比例與本地儲存鍵，而不是瞎猜樣式失效的原因。

本指南概述了開發人員如何從頭手動重建相遇卡 v19 應用程式、將其編譯為單個獨立 HTML 釋出版，並執行迴歸測試。

相遇卡使用 **React**、**TypeScript** 與 **Vite** 建置。在工作區執行以下指令初始化專案：

4. Standalone Export Compilation 4. 獨立單檔編譯匯出

To compile the modular React source code into a self-contained, single-file HTML release (like encounter_cards_v19.html), we use a custom NodeJS build finalizer script:

若要將模組化 React 原始碼編譯為獨立的單一 HTML 檔案（如 encounter_cards_v19.html），我們使用自訂的 NodeJS 建置處理器腳本：

4. 神祇 WebP 影像會轉換為 Base64 Data URL 並直接寫入程式中。
5. 產出單一的 HTML 檔案，能在任何地方離線啟動。

A complete reference guide explaining programming syntax, React hooks, TypeScript configurations, browser interfaces, and CSS 3D rendering structures for beginner developers.

📦 ES6 Modules (import & export)ES6 模組機制 (import 與 export)

In JavaScript and TypeScript, code is modularized into files. Modules let you share code between files using import and export.

在現代 JavaScript 和 TypeScript 中，程式碼會被拆分到不同的檔案中（稱為模組）。我們使用 import 與 export 來共享程式碼。

What is "import { ... }" (Curly Braces)?什麼是「import { ... }」（花括號）？

When you see curly braces like import { useEffect, useState } from 'react', it is called a Named Import (具名導入). It means we are extracting only specific tools from the 'react' package. If a package has a Default Export (預設匯出), like import App from './App', we do not use curly braces.

當您看到帶有花括號的 import { useEffect, useState } from 'react' 時，這稱為 具名導入 (Named Import)。這代表我們僅從 'react' 套件中提取特定的工具。如果是 預設匯出 (Default Export)，例如 import App from './App'，則不需要加花括號。 

// 1. Named Export (in lib/game.ts)
export function eligibleCards(...) { ... }
// Imported elsewhere as:
import { eligibleCards } from './lib/game'

// 2. Default Export (in App.tsx)
export default function App() { ... }
// Imported elsewhere as:
import App from './App'

⚛️ React Components & JSX/TSXReact 元件與 JSX/TSX 語法

React builds user interfaces by combining small, reusable building blocks called Components.

React 藉由組合微小且可重複使用的建置區塊（稱為 元件 Component）來開發使用者介面。

Understanding JSX/TSX Expressions:理解 JSX/TSX 表達式：

JSX (JavaScript XML) allows us to write HTML-like tags directly inside JavaScript. A TSX file is simply a JSX file containing TypeScript types. Inside JSX, any code wrapped in single curly braces { ... } is evaluated as active JavaScript. For example: <p>{card.zh}</p> prints the Chinese text of the card.

JSX 允許我們直接在 JavaScript 中撰寫類似 HTML 的標籤。而 TSX 則是加上了 TypeScript 型別標記的 JSX 檔案。在 JSX 中，被單花括號 { ... } 包裹的內容會被當作 JavaScript 運算。例如：<p>{card.zh}</p> 會輸出卡牌的中文問題。 

// A basic React component returning JSX
function CardText({ card, language }) {
  // Curly braces evaluate conditional logic inside the layout!
  return (
    <div>
      {language === 'zh' ? <p>{card.zh}</p> : <p>{card.en}</p>}
    </div>
  )
}

💾 useState (State Management)useState (元件狀態管理)

useState is a React Hook that lets a component "remember" changing data (state). When state changes, React automatically redraws (re-renders) the screen.

useState 是一個 React Hook，用來讓元件「記住」會改變的資料（狀態）。當狀態改變時，React 會自動重新繪製（重新渲染）畫面。

Syntax Breakdown:語法拆解說明：

const [language, setLanguage] = useState('zh')
• language: The read-only state variable.
• setLanguage: The setter function. To change the state, you must call this function: setLanguage('en').
• 'zh': The default starting value.
• Updater Initializer: If you pass a function to useState(() => loadLanguage()), React runs this function only once when the page loads, which is extremely useful to avoid reading LocalStorage repeatedly.

const [language, setLanguage] = useState('zh')
• language：唯讀的狀態變數。
• setLanguage：變更狀態的設定函數。您必須呼叫此函數如 setLanguage('en') 來改變狀態。
• 'zh'：初始預設值。
• 狀態初始化器 (Lazy Initializer)：若傳入函式例如 useState(() => loadLanguage())，React 只會在初次加載時執行它一次，這能避免元件重複讀取 LocalStorage 造成效能開銷。

⚡ useEffect (Side-Effects Lifecycle)useEffect (處理生命週期與副效能)

useEffect lets you synchronize your component with external systems (like listening to viewport resize, registering browser events, or checking service workers).

useEffect 讓元件能夠與外部系統進行同步（例如監聽螢幕寬高、註冊鍵盤事件、或與 Service Worker 離線快取同步）。

Syntax Breakdown:語法拆解說明：

useEffect(() => { ... return () => { ... } }, [dep])
• The Callback: The code inside the arrow function runs after the render is painted to the screen.
• Cleanup Function: The returned function return () => { ... } runs when the component is removed, preventing memory leaks (e.g. removing window event listeners).
• Dependency Array [dep]: Tells React when to run this effect. If empty [], the code runs exactly once when the page is loaded (mounted) and the cleanup runs once when the page is closed (unmounted).

useEffect(() => { ... return () => { ... } }, [dep])
• 回呼函式：在大括號內編寫的程式碼，會在瀏覽器繪製畫面後執行。
• 清除函式 (Cleanup)：return () => { ... } 內寫的程式碼會在元件被銷毀時執行，用來移除監聽以防記憶體洩漏。
• 依賴陣列 []：控制何時重新執行。傳入空陣列 [] 代表僅在網頁初次加載（掛載）時執行一次，網頁關閉時執行一次清除。

🧠 useMemo (Performance Caching)useMemo (快取昂貴的計算結果)

React components re-run from top-to-bottom on every render. If you have a list-filtering calculation, it will re-run constantly. useMemo caches the calculated result.

React 的元件函數在每次重新渲染時，都會從頭到尾重新執行。如果裡面有迴圈篩選，效能會大打折扣。useMemo 用於快取計算結果。

Syntax Breakdown:語法拆解說明：

const pool = useMemo(() => eligibleCards(cards, level, mode), [level, mode])
• The Formula: Calculates the initial filtered cards array.
• The Dependencies [level, mode]: React checks if either variable changed. If not, it skips recalculation and immediately returns the cached array!

const pool = useMemo(() => eligibleCards(cards, level, mode), [level, mode])
• 計算公式：執行卡牌篩選，回傳候選池。
• 依賴陣列 [level, mode]：React 會檢查這兩個參數。如果使用者只是在輸入框打字而沒改等級或模式，React 就會跳過計算，直接從快取回傳上次排好的卡牌池！

📌 useRef (Non-Reactive References)useRef (非響應式持久參照)

useRef lets you reference a value that persists across renders, but changing it does NOT force React to redraw the screen.

useRef 用於保存一個在元件重新渲染時依然能持續存在的值，但變更它時「不會」觸發 React 重新繪製畫面。

Why not just use useState?為什麼不直接用 useState？

In Encounter Cards, dragStart = useRef<number | null>(null) tracks the horizontal pixel location where a user touches the screen to drag a card. If we used state for this, every single pixel movement during a swipe would trigger a screen redraw, causing terrible lag.

在相遇卡中，dragStart = useRef<number | null>(null) 用於記錄使用者手指觸碰螢幕拉動卡牌時的起點像素。如果使用 state 紀錄，滑動時每移動一像素都會強迫螢幕重新渲染，導致嚴重的卡頓。

🏷️ TypeScript Union Types & GenericsTypeScript 聯集型別與泛型

TypeScript extends JavaScript by adding strict compile-time types to prevent runtime crashes.

TypeScript 藉由在編譯期加上嚴格的型別校驗來擴展 JavaScript，以避免執行期出錯。

Unions & Generics Explained:聯集型別與泛型解釋：

• Union Type (聯集型別): type Language = 'zh' | 'en' | 'bilingual'. A variable of type Language can only hold one of these exact strings. Typos like 'chinese' will trigger an error immediately!
• Generics (泛型): Represented by angle brackets <T>. In useState<Language>(), we tell React that this state is restricted to the Language type contract.

• 聯集型別 (Union Type)：type Language = 'zh' | 'en' | 'bilingual'。這限制變數只能存放這三種字串之一。拼寫錯誤如 'chinese' 會在編輯器中立刻標記為紅色報錯！
• 泛型 (Generics)：以角括號 <T> 表示。在 useState<Language>() 中，這告訴 React 此狀態被嚴格綁定在 Language 型別規範下。

💅 CSS Custom Properties & 3D TransformsCSS 變數與 3D 空間翻轉特效

Modern styles use CSS custom properties (variables) to enable runtime theme modifications and coordinate calculations.

現代樣式表使用 CSS 自訂屬性（變數）來實現即時主題抽換與縮放坐標運算。

CSS Custom Variables & 3D Cards:CSS 變數與 3D 立體卡牌：

• CSS Variables: Defined with double hyphens like --gold: #caa75d. React dynamically updates style={{ '--font-scale': fontScale }} to scale text sizing instantly.
• 3D Flip Effect: perspective: 1200px defines 3D space depth. transform: rotateY(180deg) rotates card elements along the Y-axis. backface-visibility: hidden hides the backside of elements when flipped!

• CSS 變數：以雙橫線開頭如 --gold: #caa75d。React 可動態修改 style={{ '--font-scale': fontScale }} 讓字型比例立刻全局生效。
• 3D 立體翻牌：perspective: 1200px 定義了 3D 空間景深。transform: rotateY(180deg) 沿 Y 軸旋轉卡牌。backface-visibility: hidden 則讓卡牌旋轉到背面時自動隱藏該面的圖文！
